- **Target Platforms**: PC (Steam) — mobile post-launch
- **Input Methods**: Mouse, Touch (future)
- **Primary Input**: Mouse (node-based tap-to-move)
- **Gamepad Support**: None
- **Touch Support**: Partial (architecture supports it, not implemented in initial release)
- **Platform Notes**: All UI must be mouse-clickable with touch-friendly hit targets for future mobile port. No hover-only interactions.

## Naming Conventions

- **Classes**: PascalCase (e.g., `PlayerController`)
- **Variables**: snake_case (e.g., `move_speed`)
- **Signals/Events**: snake_case past tense (e.g., `health_changed`, `door_opened`)
- **Files**: snake_case matching class (e.g., `player_controller.gd`)
- **Scenes/Prefabs**: PascalCase matching root node (e.g., `PlayerController.tscn`)
- **Constants**: UPPER_SNAKE_CASE (e.g., `MAX_HEALTH`)

## Performance Budgets

- **Target Framerate**: 60fps
- **Frame Budget**: 16.6ms
- **Draw Calls**: 200 (2D village scene target)
- **Memory Ceiling**: 500MB

## Testing

- **Framework**: GUT (Godot Unit Testing)
- **Minimum Coverage**: 60% (gameplay logic; not applied to visual/feel code)
- **Required Tests**: Auto-battler resolution logic, loop persistence/save state, relationship tier calculations
